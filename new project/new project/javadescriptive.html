Syntax

Class Declaration: Java programs are structured as classes which serve as blueprints for objects.

Main Method: The entry point of a Java application is the public static void main(String[] args) method.

Identifiers: Names for classes, variables, and methods, which must start with a letter, $, or _.,

Keywords: Reserved words like class, public, static, and void with predefined meanings.

Data Types: Includes primitive types (e.g., int, boolean) and non-primitive types (e.g., arrays, classes).

Operators: Include arithmetic, relational, logical, and assignment operators for various operations.

Control Statements: Direct program flow using if, else, switch, loops (for, while), and branching (break, continue).

Methods: Define behaviors with declarations that can accept parameters and return values.

Inheritance: Allows a class to inherit properties and methods from another class.

Exception Handling: Manages runtime errors using try, catch, and finally blocks, and the throw keyword.


Comments

Single-Line Comments: Use // to mark the beginning of a comment, which extends to the end of the line.

Multi-Line Comments: Enclosed between /* and */, spanning multiple lines.

Documentation Comments: Use /** and */ to create comments that can be processed by Javadoc to generate API documentation.

Purpose: Comments are used to explain code, making it easier to understand and maintain.

Single-Line Example: // This is a single-line comment.

Multi-Line Example: /* This is a multi-line comment that spans several lines */.

Documentation Example: /** This is a documentation comment for describing a class or method */.

Placement: Comments can be placed anywhere in the code, either on their own line or at the end of a line of code.

Ignored by Compiler: Comments are not executed by the compiler; they are purely for human readers.

Best Practices: Use comments to clarify complex logic, describe the purpose of variables, methods, and classes, and provide additional context where necessary.

Variables

Definition: Variables are containers used to store data in a Java program, identified by a unique name.

Declaration: In Java, variables must be declared with a specific data type before they can be used.

Data Types: Java supports primitive data types (e.g., int, double) and reference data types (e.g., String, arrays).

Syntax: Variable declaration follows the pattern: data_type variable_name;.

Initialization: Variables can be initialized at the time of declaration or later in the code.

Scope: Variables have a scope that defines where they can be accessed within the program.

Lifetime: Variables exist within their defined scope and are destroyed when the scope ends.

Naming Conventions: Variable names should be meaningful, following camelCase naming conventions.

Immutable and Mutable: Some data types, like primitives, are immutable (cannot be changed), while others, like objects, are mutable (can be modified).



Usage: Variables are used to store and manipulate data, facilitating dynamic behavior and computation in Java programs.

Operators

Arithmetic Operators: Perform basic arithmetic operations such as addition (+), subtraction (-), multiplication (*), division (/), and modulus (%).

Relational Operators: Compare values and include operators like equal to (==), not equal to (!=), greater than (>), less than (<), greater than or equal to (>=), and less than or equal to (<=).

Logical Operators: Combine boolean expressions and include logical AND (&&), logical OR (||), and logical NOT (!).

Assignment Operators: Assign values to variables and include simple assignment (=) as well as compound assignment operators like +=, -=, *=, and /=.

Increment and Decrement Operators: Increase (++), or decrease (--), the value of a variable by 1.

Conditional Operator (Ternary Operator): A shorthand for if-else statements, in the form of condition ? expression1 : expression2.

Bitwise Operators: Manipulate individual bits of integer operands and include bitwise AND (&), bitwise OR (|), bitwise XOR (^), left shift (<<), and right shift (>>).

Instanceof Operator: Checks whether an object is an instance of a particular class or interface.

Equality Operators: Test for equality (==) and inequality (!=) between two operands.

Precedence and Associativity: Operators have precedence and associativity rules that determine the order in which they are evaluated within expressions.

Strings

Definition: In Java, strings are sequences of characters, represented by the String class.

Immutable: Strings are immutable, meaning their values cannot be changed once they are created.

Initialization: Strings can be initialized using double quotes (e.g., "Hello").

Concatenation: Strings can be concatenated using the + operator (e.g., "Hello, " + "World").

Methods: The String class provides methods for various string operations like length(), substring(), indexOf(), etc.

Math

java.lang.Math: Java provides the Math class in the java.lang package for mathematical operations.

Static Methods: Math contains static methods for common mathematical operations such as trigonometric, exponential, logarithmic functions, and rounding.

Constants: It defines constants like PI and E for mathematical calculations.

No Constructor: Math class cannot be instantiated as it has a private constructor, and all its methods are static.

Utility: Used extensively in Java programs for computations ranging from simple arithmetic to complex mathematical operations.

Booleans

Definition: Booleans are a data type in Java representing true or false values.

Primitive Type: Booleans are a primitive data type in Java, denoted by the keyword boolean.

Binary Logic: Booleans are essential for binary logic operations and conditional statements in programming.

True and False: They can only have two possible values: true or false.

Used in Control Structures: Booleans are widely used in control structures such as if statements, loops, and logical operators for decision-making in programs.

If Else

Conditional Statements: if and else are control flow statements used for decision-making in Java.

if Statement: Evaluates a boolean expression and executes a block of code if the condition is true.

else Statement: Optionally follows an if statement and executes a block of code if the if condition is false.


if(condition) {
// code block to execute if condition is true
}
else {
// code block to execute if condition is false
}



Chaining: Multiple else-if blocks can be chained to handle multiple conditon

Switch

Switch Statement: Provides an alternative way to handle multiple conditions in Java.

Syntax: Uses a selector expression to match a value against multiple cases and execute corresponding code blocks.

Usage: Improves readability and maintainability for code with multiple conditional branches compared to nested if-else statements.

While loop

While Loop: A control flow statement in Java used to repeatedly execute a block of code as long as a specified condition is true.

Syntax:

while (condition) {

// code block to execute while condition is true }

Execution: The condition is evaluated before each iteration. If it evaluates to true, the code block is executed; otherwise, the loop terminates.

For loop

For Loop: A control flow statement in Java used for iterating over a range of values or elements in an array or collection.

Syntax:

for (initialization; condition; update) {

// code block to execute for each iteration }



Initialization: Executes once before the loop starts and initializes the loop control variable.

Condition: Checked before each iteration. If true, the loop continues; otherwise, it terminates.

Update: Executed after each iteration and typically updates the loop control variable.

Arrays

Arrays: Arrays are data structures in Java used to store multiple values of the same data type under a single name.

Declaration: Arrays are declared using square brackets [], specifying the data type and the size of the array.

Initialization: Arrays can be initialized with values at the time of declaration or later using assignment statements.

Indexed Access: Elements of an array are accessed using index values starting from 0.

Fixed Size: Arrays have a fixed size once they are initialized, and the size cannot be changed during runtime.

Break/Continue

Break Statement: Used in loops (e.g., for, while) and switch statements to terminate the loop or switch block prematurely.

Usage: When encountered, the break statement causes the immediate exit from the loop or switch block.

Example:

for (int i = 0; i < 10; i++) {

if (i == 5) {

break; // exits the loop when i equals 5

}

System.out.println(i); }



Continue Statement: Used in loops to skip the rest of the current iteration and proceed to the next iteration.

Usage: When encountered, the continue statement jumps to the next iteration of the loop without executing the remaining code in the loop body.

Example:

for (int i = 0; i < 10; i++) {

if (i == 5) {

continue; // skips the rest of the iteration when i equals 5

}

System.out.println(i);

}

Methods

Methods: In Java, methods are functions defined within classes to perform specific tasks.

Usage: Methods encapsulate reusable blocks of code, promoting modularity and code organization in Java programs.

Method parameter

Method Parameters: Inputs provided to a method to customize its behavior or provide data for computation.

Usage: Parameters are declared within method parentheses and can be used inside the method body to perform operations or calculations.

Method overloading

Method Overloading: A feature in Java that allows a class to have multiple methods with the same name but different parameter lists.

Usage: Enables the creation of methods that perform similar tasks but operate on different types or numbers of parameters, enhancing code readability and flexibility.

Java scope

Scope in Java: Refers to the visibility and lifetime of variables and methods within a program.

Local Scope: Variables declared inside a method or block are accessible only within that method or block.

Class Scope: Variables declared at the class level (outside any method) are accessible to all methods within the class.

Instance Scope: Instance variables belong to specific instances of a class and are accessible to all methods within that instance.

Static Scope: Static variables belong to the class itself and are shared among all instances of the class, accessible without creating an object.

Recusion

Recursion: A programming technique where a function calls itself directly or indirectly to solve a problem.

Base Case: A condition in a recursive function that stops the recursive calls and provides a termination point.

Usage: Recursion is used to solve problems that can be broken down into smaller, similar subproblems.

Example: Computing factorial or Fibonacci series, traversing trees or graphs, and solving problems like Tower of Hanoi.

Caution: Infinite recursion can lead to stack overflow errors, so ensuring a base case is crucial for termination.

Java Classes and Objects

Classes: Templates or blueprints for creating objects in Java, defining their properties and behaviors.

Objects: Instances of classes, representing real-world entities, with their own state and behavior.

Encapsulation: Classes encapsulate data (fields) and methods to manipulate that data, providing abstraction and modularity.

Instantiation: Objects are created from classes using the new keyword, allocating memory and initializing fields.

Usage: Classes and objects facilitate code organization, reuse, and maintenance in object-oriented programming paradigms.
